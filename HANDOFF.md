# HANDOFF.md — 에이전트 간 인계 프로토콜
> 최선화닷컴 프로젝트 | 모든 에이전트가 읽고 쓰는 공유 파일

---

## 📋 인계 규칙

작업이 완료되거나 다른 파트가 필요할 때:

1. **이 파일의 해당 섹션을 업데이트**한다
2. **그룹 채팅에 멘션**을 걸어 알린다
3. **다음 담당자는 이 파일을 확인**하고 작업을 이어받는다

---

## 🔄 현재 진행 상태

| 파트 | 담당 | 상태 | 마지막 업데이트 |
|------|------|------|----------------|
| 기획 (PRD) | @po_cshdotcom_bot | ✅ 완료 | 최선화닷컴_상세기획서_v1.md |
| 아키텍처 | @dev_cshdotcom_bot | ✅ 완료 | types, middleware, lib 세팅 |

| 프론트엔드 | @frontend_cshdotcom_bot | ⏳ 대기 | 컴포넌트 개발 시작 필요 |
| QA | @qa_cshdotcom_bot | 🔄 진행 중 | 테스트 인프라 완료, E2E 페이지 구현 대기 중 |

## 📬 인계 큐 (미완료 핸드오프)

### [BACKEND → FRONTEND] API 라우트 준비 완료 시
```
백엔드가 API 라우트 완성하면:
- 어떤 엔드포인트가 사용 가능한지 이 파일에 업데이트
- @frontend_cshdotcom_bot 멘션: "API 준비됨, 연동 시작해줘"
```

### [FRONTEND → QA] 컴포넌트/페이지 구현 완료 시
```
프론트엔드가 페이지 구현하면:
- 구현된 라우트 목록 이 파일에 업데이트
- @qa_cshdotcom_bot 멘션: "XXX 페이지 구현됨, E2E 돌려줘"
QA는 E2E 테스트 실행 → 결과 이 파일에 기록
```

### [QA → FRONTEND/BACKEND] 버그 발견 시
```
QA가 버그 발견하면:
- 아래 버그 리포트 섹션에 기록
- 담당 파트 멘션으로 알림
```

### [ALL → DEV] 블로커 발생 시
```
어느 파트든 막히면:
- 블로커 섹션에 기록
- @dev_cshdotcom_bot 멘션으로 에스컬레이션
```

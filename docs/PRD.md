## 1. 서비스 개요

### 1.1 목적
강연 기획사 "최선화닷컴"의 공식 웹 플랫폼. 강사와 잠재 고객(기업/기관)을 연결하고, 강연 기획 문의부터 강사 섭외까지 원스톱으로 처리하는 서비스.

### 1.2 핵심 사용자
| 구분 | 설명 |
|------|------|
| 일반 방문자 | 강사 탐색, 강연 문의 |
| 회원 | 북마크, 히스토리, 맞춤 추천 |
| 강사 | 프로필 등록 요청 |
| 어드민 | 전체 콘텐츠/문의/회원 관리 |

### 1.3 URL 구조
```
최선화닷컴/                  ← 메인 사이트
최선화닷컴/mong-bab/         ← 어드민 (비공개 URL)
최선화닷컴/mong-bab/login    ← 어드민 로그인
```

## 3. 메인 사이트 상세 기능 명세

---

### 3.1 회원가입 / 로그인

#### 3.1.1 이메일 회원가입

**가입 폼 필드**
- 이름 (필수, 최대 20자)
- 이메일 (필수, 중복 확인)
- 비밀번호 (필수, 8자 이상, 영문+숫자+특수문자 1개 이상)
- 비밀번호 확인 (필수, 일치 검증)
- 마케팅 수신 동의 (선택, 체크박스)
- 이용약관 동의 (필수)
- 개인정보 처리방침 동의 (필수)

**가입 플로우**
1. 폼 제출 → 이메일 인증 메일 발송
2. 이메일 내 링크 클릭 → 인증 완료
3. 인증 완료 후 자동 로그인 & 홈 리디렉션

**UX 포인트**
- 실시간 이메일 중복 확인 (blur 이벤트, 디바운스 300ms)
- 비밀번호 강도 인디케이터 (약/중/강)
- 비밀번호 표시/숨기기 토글
- 에러: 필드 아래 인라인 메시지, 레드 border

**엣지케이스**
- 이미 가입된 이메일 → "이미 사용 중인 이메일입니다. 로그인하시겠어요?" + 로그인 링크
- 인증 링크 만료 (24시간) → "링크가 만료되었습니다. 재발송하시겠어요?" + 재발송 버튼
- 소셜 로그인과 동일 이메일 → 소셜 계정으로 연결 안내

---

#### 3.1.2 구글 소셜 로그인

**플로우**
1. "구글로 계속하기" 버튼 클릭
2. 구글 OAuth 2.0 팝업/리디렉션
3. 콜백 처리 → 신규면 자동 회원가입, 기존이면 로그인
4. 홈 또는 이전 페이지 리디렉션

**UX 포인트**
- 최초 구글 로그인 시 이름, 이메일 자동 채워짐
- 마케팅 수신 동의 팝업 (최초 1회)
- 버튼: 구글 공식 브랜드 가이드라인 준수

---

#### 3.1.3 로그인

**필드**
- 이메일
- 비밀번호
- 자동 로그인 (30일 쿠키)

**엣지케이스**
- 5회 로그인 실패 → 15분 잠금, 이메일 알림
- 비밀번호 찾기 → 이메일 발송, 1시간 유효 링크
- 계정 미인증 → "이메일 인증이 필요합니다" + 재발송 버튼

### 3.2 강사 소개 리스트 (/speakers)

#### 3.2.1 리스트 페이지

**화면 구성**
- 상단: 필터 바 (분야, 지역, 강연료 범위)
- 중단: 검색창 (강사명, 키워드)
- 하단: 카드 그리드 (PC: 4열, 태블릿: 2열, 모바일: 1열)

**강사 카드 표시 정보**
- 프로필 사진 (없으면 기본 이미지)
- 강사명
- 주요 분야 태그 (최대 3개)
- 한 줄 소개 (최대 40자, 초과 시 말줄임)
- 대표 커리큘럼 1개
- 문의 버튼

**필터 항목**
- 분야: 리더십, 동기부여, 마케팅, 조직문화, AI/기술, 커뮤니케이션 등 (복수 선택)
- 강연료: 전체 / 100만원 미만 / 100~300만원 / 300만원 이상

**정렬**
- 기본: 추천순 (어드민 설정)
- 선택: 이름순 / 최신 등록순

**페이지네이션**
- 무한 스크롤 (모바일) / 20개씩 페이지네이션 (PC)

**빈 결과 처리**
- "해당 조건의 강사가 없습니다. 직접 강사 섭외를 문의해보세요." + 문의 CTA

---

#### 3.2.2 강사 상세 페이지 (/speakers/:id)

**섹션 구성**

① 헤더 영역
- 강사 대표 사진
- 이름, 소속/직함
- 분야 태그
- 문의 CTA 버튼 (항상 노출, 스크롤 시 fixed)
- 공유 버튼 (URL 복사 / 카카오톡)

② 강사 소개
- 약력 (텍스트 + 타임라인 UI)
- 주요 강연 이력 (기업/기관명 로고 + 회사명)
- 미디어 (유튜브 영상, 뉴스 링크)

③ 커리큘럼 목록
- 강사가 보유한 강연 커리큘럼 카드 리스트
- 각 카드: 강연명, 시간, 대상, 한 줄 설명
- 클릭 시 해당 강연 상세로 이동

④ 수강 후기 (있을 경우)
- 별점 평균
- 리뷰 카드 (기업명, 내용, 날짜)

---

### 3.3 강연 커리큘럼 (/lectures)

#### 3.3.1 강연 목록 페이지

**화면 구성**
- 상단: 카테고리 탭 (전체, 분야별)
- 검색: 강연명, 키워드
- 리스트: 강연 카드 그리드

**강연 카드 표시 정보**
- 썸네일 이미지
- 강연명
- 담당 강사 이름 (클릭 → 강사 상세)
- 강연 시간 (예: 2시간)
- 분야 태그
- 한 줄 요약

---

#### 3.3.2 강연 상세 페이지 (/lectures/:id)

**섹션 구성**

① 헤더
- 강연명
- 강사 정보 (사진 + 이름, 클릭 → 강사 상세)
- 강연 시간 / 대상 / 분야
- 문의 CTA 버튼

② 강연 개요
- 강연 목표 (3~5개 불릿)
- 강연 대상 (누구에게 적합한지)
- 프로그램 구성 (시간대별 목차)

③ 기대 효과
- 참가자가 얻어가는 것 (불릿 or 카드 형태)

④ 강사 소개 (간략, 더보기 → 강사 상세 링크)

⑤ 관련 강연 추천 (같은 분야, 최대 4개)

### 3.4 문의하기 (/inquiry)

#### 3.4.1 문의 유형 선택
첫 화면에서 2개 카드로 분기:
- 강연 기획 / 강사 섭외 문의
- 강사 등록 문의

---

#### 3.4.2 강연기획 / 강사섭외 문의

**폼 필드**
- 문의 유형 (강연기획 / 강사섭외) [라디오]
- 담당자 이름 (필수)
- 연락처 (필수, 형식 검증)
- 이메일 (필수)
- 소속 기관/회사명 (필수)
- 원하는 강사 또는 분야 (선택, 자유 입력)
- 강연 예정일 (선택, 날짜 피커)
- 예상 인원수 (선택, 숫자 입력)
- 강연 장소 (선택, 자유 입력)
- 예산 범위 (선택, 드롭다운)
- 문의 내용 (필수, 최대 1000자, 글자 수 카운터)
- 개인정보 수집/이용 동의 (필수)

**UX 포인트**
- 강사 상세 페이지에서 "문의하기" 클릭 시 해당 강사 정보 자동 채워짐
- 제출 후: "문의가 접수되었습니다. 1~2 영업일 내 연락드리겠습니다." 확인 페이지
- 접수 확인 이메일 자동 발송 (입력한 이메일로)

---

#### 3.4.3 강사 등록 문의

**폼 필드**
- 강사 이름 (필수)
- 연락처 (필수)
- 이메일 (필수)
- 소속/직함 (필수)
- 주요 강연 분야 (복수 선택, 최대 3개)
- 보유 강연 주제 (필수, 자유 입력)
- 주요 강연 이력 (선택, 자유 입력)
- 프로필 자료 첨부 (선택, PDF/PPT/Word, 최대 10MB)
- 개인정보 수집/이용 동의 (필수)

## 4. 어드민 (/mong-bab/) 상세 기능 명세

### 4.0 어드민 공통

**접근 방식**
- /mong-bab/ URL은 공개하지 않음 (노출은 하되 일반 사용자가 모름)
- /mong-bab/login → 어드민 전용 이메일/비밀번호 로그인
- 로그인 없이 접근 시 → /mong-bab/login 리디렉션
- 세션 만료: 4시간 비활동 시 자동 로그아웃

**공통 레이아웃**
- 좌측: 사이드바 메뉴 (대시보드, 회원, 강사, 강연, 문의, 고객지원)
- 상단: 브레드크럼 + 어드민 이름 + 로그아웃
- 우측: 콘텐츠 영역

## 5. 비기능 요구사항

### 5.1 반응형 / 접근성
- 브레이크포인트: 320px / 768px / 1280px
- 모든 인터랙티브 요소 키보드 접근 가능
- 이미지 alt 텍스트 필수

### 5.2 SEO
- 강사 상세, 강연 상세: 메타 태그 (title, description, og:) 동적 생성
- sitemap.xml 자동 생성

### 5.3 성능
- LCP 목표: 2.5초 이하
- 이미지: WebP 변환 + lazy loading
- 강사 리스트: 서버사이드 페이지네이션

### 5.4 보안
- 어드민 URL(/mong-bab/) 외부 노출 최소화 (robots.txt disallow)
- 파일 업로드: 타입 화이트리스트 + 크기 제한
- API 인증: JWT + refresh token

## 7. 개발 우선순위 (Phase)

### Phase 1 (MVP) — 핵심 공개 기능
1. 강사 리스트 + 상세

3. FAQ, 공지사항, 회사소개
4. 어드민: 회원 관리, FAQ/공지 관리

### Phase 3 — 고도화
1. 강사 추천 알고리즘
2. 회원 북마크
3. 문의 처리 현황 이메일 알림
4. 대시보드 통계 고도화
